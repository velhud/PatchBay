# OSS Pipeline

`patchbay` is the first public repository in a staged OSS direction around ChatGPT-to-local-Codex bridging and controlled Codex/MCP maintainer infrastructure.

## Current Public Focus

- Streamable HTTP MCP PatchBay for ChatGPT, local workspace context, and Codex CLI workflows.
- Local-control ChatGPT-to-Codex automation.
- Read-only repository analysis.
- Isolated worktree-based apply jobs.
- Status, result, diff, review, and resume workflows.
- `.ai-bridge` handoff and Pro-style context pack workflows.

## Planned Staged Releases

The following categories are being cleaned before public release:

- contextual memory and evidence-bundle formats;
- agent supervisor/verifier control-plane templates;
- MCP connector templates for tools and providers;
- provider-routing and unified AI access examples;
- OSS search and repository-discovery workflows;
- ChatGPT/Codex maintainer workflow examples for PR review, issue triage, release notes, documentation, and security review.

## API Credit Use

API credits will be used only for open-source ChatGPT/Codex bridge development and maintainer workflows:

- PR review and test generation;
- release notes and documentation;
- MCP tool-description evaluation;
- agent-memory retrieval evaluation;
- planner, worker, and verifier workflow evaluation;
- public examples and reproducible demos.

## Security Scope

Codex Security will be used only on repositories I own, maintain, or am authorized to administer.

Primary focus:

- MCP server attack surface;
- authentication and token handling;
- local and remote access boundaries;
- command execution and worktree isolation;
- path validation;
- secret handling;
- PR/diff security review before release.
